Step-by-Step Guide: Hosting via GitHub Pages & Setting up `robot.local`

Since your dashboard code runs locally in the browser and connects via MQTT (WebSocket) and your ESP32's IP/mDNS stream, hosting it on GitHub Pages is super easy. However, remember that because GitHub Pages uses HTTPS, your browser might block an insecure `http://` video stream URL. The best workaround is either serving the HTML directly from the ESP32 (SPIFFS/LittleFS) or using a local browser workaround.

Phase 1: Uploading to GitHub Pages

  1. Go to GitHub and create a new repository (e.g., named robot-hud). Make it Public.
  2. Rename your final HTML file to exactly index.html.
  3. Upload your index.html file (and any local images if you aren't using URLs) into the main branch of your repository.
  4. Go to your repository's Settings tab at the top.
  5. In the left sidebar, click on Pages.
  6. Under Build and deployment -> Branch, select main (or master) and `/ (root)`, then click Save.
  7. Within a minute, GitHub will give you a live URL like `https://yourusername.github.io/robot-hud/`.

Phase 2: Achieving the `robot.local` URL

To use an easy-to-remember address like http://robot.local instead of a raw IP address, your ESP32 needs to broadcast its hostname on your local Wi-Fi network using mDNS.

  1. Ensure your ESP32 code includes the mDNS library and initializes it right after connecting to Wi-Fi: 
    #include <ESPmDNS.h>

// Inside setup():
if (!MDNS.begin("robot")) {
    Serial.println("Error setting up MDNS responder!");
} else {
    Serial.println("mDNS responder started: http://robot.local");
}
  2. Once your ESP32 boots up and connects to your router, any device (phone, laptop, PC) connected to that same Wi-Fi network can simply type **`http://robot.local`** into the browser to access the ESP32's local server.

Pro-Tip for Mixed Content (HTTPS vs HTTP)

If you host your HUD on GitHub Pages (which is HTTPS) but try to pull an unencrypted video stream (`http://192.168.1.100:81/stream`), modern browsers will block it for security ("Mixed Content Error"). To avoid this completely, you can save your index.html file directly onto the ESP32's flash memory (LittleFS) or use a local helper setup so everything runs natively under the exact same domain!
